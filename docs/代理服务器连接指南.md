# 代理服务器连接指南

本文面向第一次使用本工具包的读者，说明客户端如何连接微信公众号代理服务器。

## 1. 为什么需要代理服务器

微信公众号接口通常要求稳定的服务端网络环境，并涉及 AppID、AppSecret、接口调用凭据及 IP 白名单。为了避免把这些敏感信息分发到每位使用者的电脑，本工具包通过一个独立的 HTTPS 代理服务完成素材上传和草稿管理。

调用链如下：

```text
本地排版工具 → HTTPS + Basic Auth → 代理服务器 → 微信公众号接口 → 草稿箱
```

本仓库只包含本地客户端和接口约定，不包含可直接部署的代理服务端，也不提供公共代理账号。请向你的社团、团队管理员或代理服务维护者获取接入信息。

## 2. 连接前需要准备什么

请向代理服务维护者索取：

1. HTTPS 根地址，例如 `https://proxy.example.com`；
2. Basic Auth 用户名；
3. Basic Auth 密码；
4. 自签名证书对应的 CA 证书文件（仅在服务未使用公共可信证书时需要）。

代理服务器维护者还需确保：

- 服务端已配置可用的微信公众号 AppID 与 AppSecret；
- 服务端出口 IP 已加入微信公众号后台的 IP 白名单；
- HTTPS 证书有效，且没有要求客户端关闭 TLS 校验；
- 上传大小、请求频率和账号权限满足实际使用需求。

## 3. 配置环境变量

### Windows PowerShell

以下配置仅在当前 PowerShell 窗口有效，关闭窗口后不会保留：

```powershell
$env:MP_PROXY_URL = "https://proxy.example.com"
$env:MP_PROXY_USERNAME = "publisher"
$env:MP_PROXY_PASSWORD = "请替换为真实密码"
$env:MP_PROXY_CA_CERT = "C:\证书\mp-proxy-ca.crt"
```

若代理使用受操作系统信任的公共证书，可以不设置 `MP_PROXY_CA_CERT`：

```powershell
Remove-Item Env:MP_PROXY_CA_CERT -ErrorAction SilentlyContinue
```

### Linux / macOS

```bash
export MP_PROXY_URL="https://proxy.example.com"
export MP_PROXY_USERNAME="publisher"
export MP_PROXY_PASSWORD="请替换为真实密码"
export MP_PROXY_CA_CERT="/绝对路径/mp-proxy-ca.crt"
```

请勿把真实密码、CA 私钥或含有凭据的 `.env` 文件提交到 GitHub。CA 证书本身通常不含私钥，但仍应按团队安全规范保存。

## 4. 测试连接

先安装依赖：

```bash
python -m pip install -r requirements.txt
```

然后执行健康检查：

```bash
python -c "from mp_proxy import RemoteMP; mp=RemoteMP(); print(mp.health())"
```

正常情况下会返回 JSON，且 `ok` 为 `true`。也可以用 Python 脚本测试：

```python
from mp_proxy import RemoteMP

with RemoteMP() as mp:
    result = mp.health()
    print(result)
```

如需绕过环境变量，可显式传参，但不建议把密码直接写入代码：

```python
from mp_proxy import RemoteMP

with RemoteMP(
    base_url="https://proxy.example.com",
    auth=("publisher", "请替换为真实密码"),
    ca_cert=r"C:\证书\mp-proxy-ca.crt",
) as mp:
    print(mp.health())
```

## 5. 创建第一篇草稿

准备：

- 一份 UTF-8 编码的 HTML 正文；
- 一张封面图；
- 正文中用到的图片。

HTML 中使用语义化占位符：

```html
<section>
  <p>这里是文章正文。</p>
  <img src="{{IMAGE:activity-photo}}" alt="社团活动合影">
</section>
```

运行：

```bash
python tools/publish_draft.py \
  --title "示例文章" \
  --html article.html \
  --cover cover.jpg \
  --image activity-photo=activity.jpg \
  --author "社团名称"
```

Windows PowerShell 可使用反引号换行：

```powershell
python tools\publish_draft.py `
  --title "示例文章" `
  --html article.html `
  --cover cover.jpg `
  --image activity-photo=activity.jpg `
  --author "社团名称"
```

工具会依次检查代理健康状态、上传封面和正文图片、替换图片占位符、创建草稿，并回读草稿进行核验。它不会群发文章。

## 6. 代理接口约定

兼容的代理服务端需要实现以下接口，并使用 JSON 响应。客户端会在已配置用户名和密码时通过 HTTP Basic Auth 发送凭据。

| 方法 | 路径 | 用途 | 成功响应关键字段 |
|---|---|---|---|
| `GET` | `/health` | 检查服务状态 | `{"ok": true}` |
| `POST` | `/upload-thumb` | 上传封面图，表单字段名为 `file` | `media_id` |
| `POST` | `/upload-inline` | 上传正文图片，表单字段名为 `file` | `url` |
| `POST` | `/draft-add` | 创建草稿 | `draft_media_id` |
| `POST` | `/draft-get` | 获取草稿 | `news_item` |
| `POST` | `/draft-update` | 更新草稿 | 成功 JSON |
| `POST` | `/draft-delete` | 删除草稿 | 成功 JSON |

创建草稿的请求结构示例：

```json
{
  "articles": [
    {
      "title": "文章标题",
      "content": "<p>正文</p>",
      "thumb_media_id": "封面素材 ID",
      "author": "作者",
      "digest": "摘要"
    }
  ]
}
```

代理服务应返回明确的 HTTP 状态码和 JSON 错误信息，并在服务端安全地管理微信公众号凭据。客户端默认请求超时为 120 秒。

## 7. 常见问题

### 提示 `Missing proxy URL`

没有设置 `MP_PROXY_URL`，或当前终端没有继承该变量。重新设置环境变量后再运行。

### 返回 HTTP 401

用户名或密码错误，也可能是代理未启用预期的 Basic Auth。请向维护者核对账号，不要把密码粘贴到公开 Issue。

### 出现 TLS 或证书错误

确认 `MP_PROXY_CA_CERT` 指向正确的 CA 证书绝对路径，并检查证书是否过期、域名是否匹配。不要使用 `verify=False` 绕过校验。

### 连接超时或拒绝连接

检查代理域名、端口、防火墙及服务器运行状态。必要时请维护者确认 `/health` 是否可以从你的网络访问。

### 微信接口提示 IP 不在白名单

需要把代理服务器的固定出口 IP 加入微信公众号后台白名单；应添加代理服务器出口 IP，而不是每位使用者的本地 IP。

### 上传成功但草稿创建失败

检查封面素材类型、图片大小、正文 HTML 是否符合微信公众号限制，以及代理服务端日志中的微信错误码。

## 8. 安全建议

- 代理必须使用 HTTPS；
- 为不同团队或用途分配独立账号，并定期轮换密码；
- 对上传大小、请求频率和可调用接口设置限制；
- 服务端日志不得记录 Basic Auth 密码、AppSecret 或完整访问令牌；
- 不要把代理暴露为无需认证的公共服务；
- 不要在公开 Issue、截图或终端录屏中展示真实凭据。
